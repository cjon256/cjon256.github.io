Immutable Annoyance

So quotes like this annoy me

The immutable server pattern: Once code has been committed in Git, we don’t want anything else to change.

  • The GitOps Pipeline - Part 2

the thing is, it oversimplifies.

There is some state anytime your code runs. What they mean is immutable is configuration, (by which I mean state that persists between runs).

There are, it seems to me, three distinct sources of persistent state. Explicit configuration files, data stored in the Database and the code itself (which encodes the logic that your program carries out). Your code and config are the immutable part. The database is mutable, and mutation on it must be tightly controlled.

Transitory state is the state required by the running program and should be deterministic based on knowing the inputs (in an ideal world). But knowing all the inputs is not always trivial, and the world is seldom ideal. So often paying attention to the transitory state is the easier route. This is why good logging is essential.
