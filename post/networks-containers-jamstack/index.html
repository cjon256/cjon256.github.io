Networks

I’ve been a network admin at a small web host for over a decade. Started with Foundry, did Cisco Cat6500 for far too long, and then switched to Juniper (MX and QFX primarily). When I started I was a hybrid programmer/sysadmin/netadmin because the company was less than 10 people. Slowly my role became more and more network focused because I was the only one who really “understood that stuff.”

Well that, and other things. The clean separation between config and state in network devices always seemed much nicer and cleaner than the sysadmin environment. And getting involved with programming in that environment was less appealing because the company was PHP based, and… PHP of that era was not to my taste. It’s still one of my least favorite languages, but I do hear it is much improved.1

And then it slowly shifted back to my needing all 3 hats as we got servers to monitor, so SysAdmin again. And automation was required as we scaled up so that brought back the programmer bit.

Containers

Containers provide a standard way to package application’s code and dependencies into a single object &emdash; AWS Deep Dive Containers

But the SysAdmin world has shifted a good bit thanks to containers. The separation between config and more transitory state is much better. And the automation tools are similar to what we use on the network side.

This means that being a sysadmin is really just as clean as being a netadmin, which is a huge improvement.
